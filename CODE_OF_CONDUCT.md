# Code of Conduct

Custodia is a security-focused open-source project. Participation in this project must be professional, respectful and constructive.

## Expected behavior

- Be respectful and inclusive in issues, pull requests, reviews and discussions.
- Keep technical disagreement focused on evidence, tests, threat models and maintainability.
- Respect security boundaries: do not request or propose changes that weaken mTLS, client-side crypto, audit integrity or authorization without a clearly documented security rationale.
- Report security-sensitive findings privately according to `SECURITY.md`.

## Unacceptable behavior

- Harassment, intimidation, personal attacks or discriminatory language.
- Publishing private information, credentials, keys, exploit details or sensitive vulnerability information in public project spaces.
- Repeatedly derailing technical discussions after maintainers have asked for focus.
- Using project infrastructure to promote spam, malware or unrelated commercial content.

## Enforcement

Maintainers may moderate, edit, hide or remove comments, issues, pull requests or other contributions that violate this code of conduct. Maintainers may also block participants from project spaces when needed to protect contributors or the project.

## Reporting

For conduct concerns, contact the maintainer through the repository profile or GitHub account contact methods. For vulnerabilities or security-sensitive issues, follow `SECURITY.md` instead of opening a public issue.
